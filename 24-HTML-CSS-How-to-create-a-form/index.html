01. A form is used on a webpage so that users can input data to a login system for example



01a. We use input and set the type. Popular types are "text", "button", "checkbox", "date", "email" etc. See https://www.w3schools.com/html/html_form_input_types.asp Input type is obligatory.

01b. Name. We give the input type a name. it can be anything but I recommend only lowercase letters, no special characters and one word. if two woerds are needed then use an underscore to seperate the words as a space character is not permitted. Input name is obloÂ¡igatory.

01c. Next we have optional perameters. One example is placeholder. A placeholder is greyed out text that appears in the input box to guide the user to what they should write. this text is overwritten once the user starts to type in the box.

01d. Optional perameters. Value specifies a default value which will remain unless the user corrects it.



02. Another type of input is called radio.

02a. Note that the "name" has to match in both input statements if this is a select one or the other type. I.e. we can only chose male or female and not both.

02b. We can also define a default value to a radio input by setting one of them to checked.



03. A third type of input is called submit.

03a. This is a save button or a button that we can write PHP or other programming language to perform actions with the data that has been inputed. It is optional but advisory to set a name.



04. Now we need to tell the form what to do with the data once the submit button is clicked..

03a. We do this by setting the form action. In this case I have told it to pass the information onto a file called contactform.php.

03b. Also we need to specify the method that the form passes on the information. GET is when the values in the inputs are passed via the URL. THis is not secure. POST is when the URL is left without adding the information and is considered more secure. If a form uses passwords or other sensative information then POST must be used. Note that GET has a limit of 3000 characters inside the URL whereas POST does not.



05. We can also create an input type called password.

05a Note the the action method is set to get.

05-1 The form action is passing the contents of the form to contactform.php via the URL.

05-2 The email address is also passed in plain text via the URL.

05-3 Gender is also passed in plain text via the URL.

05-4 although the input on the form for password shows the input text as asteriks, when it is passed via the URL it is in plain text. In this case I set the password to password.

07. We can also have a textarea on a form. This is a type of input but is used with "textarea" opening and closing tags.

07a. A text area is used to enter text on multiple lines with enter and spaces etc. It can be used on blog input forms or email contact forms.

07c. This input doe not need a type specified because it is already set to text by default. Without CSS styling the text area is expandable on the HTML form using the drag icon on the bottom right corner.



08. A select is used to have a drop down selection box on an input.



09. A button tag is used to actually define a button.

09a. Where type is submit and name is whatever whe choose.

09a. we can also add the text we would like to appear on the button between the opening and closing tag.
